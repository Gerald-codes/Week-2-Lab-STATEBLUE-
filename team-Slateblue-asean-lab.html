Setup

library(tidyverse)
library(readxl)
library(waldo)

3.2 Import the ASEAN Meat Production Data

meat_raw <- read_xlsx(
  path = "ASEAN-Statistical-Yearbook-2023.xlsx",
  sheet = "X.14",
  range = "A4:L36"
)

meat_raw

We imported the relevant table from sheet X.14 using read_xlsx(). The selected range includes the animal-type column, the ten ASEAN country columns, and the ASEAN aggregate column. The aggregate column is kept during import so that the raw table remains close to the original spreadsheet, but it is removed during tidying because it is a derived total rather than an individual country observation.

3.3 Tidy the Meat Production Data

meat_tidy <- meat_raw |>
  rename(animal = `Animal Type`) |>

  mutate(
    year = str_extract(as.character(animal), "^20[0-9]{2}$"),
    year = as.integer(year)
  ) |>
  fill(year) |>

  filter(
    animal %in%
      c(
        "Beef and Buffalo Meat",
        "Pig meat",
        "Mutton and Goat Meat",
        "Poultry meat"
      )
  ) |>

  select(-ASEAN) |>

  pivot_longer(
    cols = -c(year, animal),
    names_to = "country",
    values_to = "production"
  ) |>

  mutate(
    animal = recode(
      animal,
      "Beef and Buffalo Meat" = "beef and buffalo",
      "Pig meat" = "pig",
      "Mutton and Goat Meat" = "mutton and goat",
      "Poultry meat" = "poultry"
    ),
    production = as.numeric(production)
  ) |>

  select(country, animal, year, production) |>
  arrange(desc(year), animal, country)
slice_head(meat_tidy, n = 6)
slice_tail(meat_tidy, n = 6)
nrow(meat_tidy)
[1] 160

Verifier Task: contains exactly the four product strings

unique(meat_tidy$animal)
[1] "beef and buffalo" "mutton and goat"  "pig"              "poultry"

The raw meat-production table was converted into tidy format by creating a year column from the year section rows, filling the year values downward, keeping only the four required animal categories, removing the ASEAN aggregate column, and pivoting the country columns into one country column. The animal names were standardised to lower-case labels so that they match the required values exactly.

The resulting table has one row for each country, animal type, and year. The verifier checks confirm that the table has 160 rows and that the animal column contains only the four required values.

3.4 Correct Cambodia’s Mutton-and-Goat Entries

meat_tidy_original <- meat_tidy

meat_tidy <- meat_tidy_original |>
  mutate(
    production = if_else(
      country == "Cambodia" & animal == "mutton and goat",
      NA_real_,
      production
    )
  )
meat_tidy |>
  filter(country == "Cambodia", animal == "mutton and goat")

Verifier Task

meat_tidy_verifier <- meat_tidy_original

meat_tidy_verifier[
  meat_tidy_verifier$country == "Cambodia" &
    meat_tidy_verifier$animal == "mutton and goat",
  "production"
] <- NA_real_

waldo::compare(meat_tidy, meat_tidy_verifier)
✔ No differences

Cambodia’s mutton-and-goat production values were changed from 0 to NA because the lab states that the PDF version records these entries as unavailable rather than true zero. The correction uses both country == "Cambodia" and animal == "mutton and goat" so that only the four affected rows are changed. Other zero values in the dataset are left unchanged.

The verifier used a different replacement method with base R indexing and compared the result against the Coder’s tidyverse method. Since waldo::compare() reports no differences, both approaches produce the same corrected table.

3.6 Tidy the Population Data

pop_raw <- read_excel(
  "ASEAN-Statistical-Yearbook-2023.xlsx",
  sheet = "I.1",
  skip = 4,
  n_max = 10
)
pop_tidy <- pop_raw |>
  pivot_longer(
    cols = -Country,
    names_to = "year",
    values_to = "pop"
  ) |>
  rename(
    country = Country
  ) |>
  mutate(
    country = str_trim(country),
    year = as.integer(year),
    pop = as.numeric(pop)
  ) |>
  select(country, year, pop) |>
  arrange(desc(year), country)

head(pop_tidy, 6)

The population table was reshaped from wide to long format so that each row represents one country-year observation. The year columns were gathered into a single year column, and the population values were stored in pop. The country names were cleaned using str_trim() to remove leading or trailing spaces, including the trailing space in "Malaysia ".

Verifier Task

pop_tidy |>
  filter(str_detect(country, "Malaysia")) |>
  distinct(country) |>
  mutate(
    n_char = str_length(country),
    has_trailing_space = str_detect(country, "\\s$")
  )

The trailing space is not present in pop_tidy$country because str_trim(country) removes whitespace from the beginning and end of each country name. Therefore, "Malaysia " becomes "Malaysia" before the final table is sorted and compared.

3.8 Join Meat Production to Population

merged <- meat_tidy |>
  left_join(pop_tidy, by = c("country", "year")) |>
  select(country, animal, year, production, pop)
merged

We used left_join() to preserve every row from meat_tidy while adding the matching population value from pop_tidy using both country and year as keys. This is appropriate because the meat-production table is the main dataset, and the population values are needed only to calculate per-capita production.

verifier

compare(
  merged |> select(country, animal, year, production),
  meat_tidy
)
✔ No differences

3.9 Derive Per-Capita Production

merged <- merged |>
  mutate(
    meat_per_cap = production / pop * 1000
  )

merged |>
  filter(country == "Cambodia", animal == "mutton and goat")

The new variable meat_per_cap represents meat production in kilograms per person. Since production is measured in thousand metric tons and population is measured in thousands of people, the conversion is production / pop * 1000. Rows with missing production values, such as Cambodia’s mutton-and-goat entries, remain missing after the calculation.

verifier

merged_verifier <- merged |>
  select(country, animal, year, production, pop)

merged_verifier$meat_per_cap <- merged_verifier$production /
  merged_verifier$pop *
  1000

waldo::compare(merged, merged_verifier, tolerance = 1e-6)
✔ No differences

ggplot(

3.12 Comment on Plot

One noteworthy pattern is that poultry production per person is generally much higher than mutton-and-goat production across many ASEAN countries. This can be seen from the poultry lines, which are usually above the mutton-and-goat lines in most country panels. However, these values should be interpreted as production per person rather than consumption per person, because the calculation does not account for imports, exports, or stock changes.

Source: ASEAN Secretariat (2023), ASEAN Statistical Yearbook 2023.
